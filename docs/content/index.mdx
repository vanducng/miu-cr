**miu-cr** (MIU Code Review) is a fast, pure-Go AI code-review CLI with a deterministic + agent engine. Review your own changes locally before you open a PR, gate them in CI, review GitHub PRs with inline comments, or drive the engine from any MCP-capable agent host (Claude Code, Codex, …). One review path, four ways to run it, a stable JSON envelope on stdout.

Diff-only review misses cross-file bugs; bare-agent review drifts and burns tokens. miu-cr keeps the **correctness-critical parts deterministic** — file selection, context assembly, line-anchoring, severity gating, dedupe — and uses the LLM only where judgment helps: finding bugs and proposing fixes.

:::note[v0.11.0]
Local review, GitHub PR review, project rules, the serve/poll daemon + GitHub Action, SQLite/Postgres stores, opt-in semantic code-recall, a REST API + GitHub App auth, and an MCP server all ship today.
:::

## What you get (v0.11.0)

- **Local review** — `miucr review --staged` / `--from`+`--to` / `--commit`: deterministic select → context → single LLM pass → line-anchor → gate → dedupe. Non-zero exit at/above `--gate` severity (CI-friendly). → [Usage](/usage/)
- **GitHub PR review** — `review --pr <url|owner/repo#N>`: head-SHA-anchored inline comments + one idempotent summary; content-stable cross-push dedupe + resolution; opt-in `--suggest` and `--approve-clean`. → [GitHub PR review](/github-pr/)
- **Project rules** — `.miu/cr/rules/*.md` markdown with frontmatter selects context by glob; built-in defaults + user + repo layers, trust-fenced on fork PRs. → [Project rules](/rules/)
- **serve · poll · Action** — an HMAC webhook daemon, an opt-in poll trigger, and a reusable GitHub Action — all funnel into one review path. → [Serve & Action](/serve-and-action/)
- **Store backends** — SQLite by default; opt into Postgres via `[store] backend` + `MIUCR_PG_DSN`. → [Store backends](/store-backends/)
- **Semantic code-recall** — opt-in embeddings + pgvector recall prior findings as advisory context; off by default. → [Semantic code-recall](/semantic-recall/)
- **REST API + GitHub App** — `MIUCR_API_TOKEN` registers `/v1/reviews`; `[github] mode = app` swaps PAT auth for App installation auth. → [REST API & GitHub App](/rest-api-and-github-app/)
- **MCP server** — `miucr mcp` exposes `review_run` / `review_get` over stdio to any MCP runtime. → [MCP integration](/mcp/)
- **Pure-Go** (`CGO_ENABLED=0`) single static binary; credentials are passed at runtime and **never persisted**.

## A 30-second tour

```sh
export ANTHROPIC_API_KEY=...                       # bring your own key (--provider auto detects OpenAI too)
miucr review --staged                              # review staged changes
miucr review --from main --to HEAD -o pretty       # review a range, human-readable
miucr review --commit HEAD~1 --gate high           # exit 2 if a high+ finding lands
miucr review --pr owner/repo#123 --no-post -o json # dry-run a GitHub PR
miucr mcp                                          # serve the engine over MCP (stdio)
```

Every command emits the stable `miucr.cli/v1` JSON envelope by default (`-o pretty` for a human table). New here? Start with **[Getting started](/onboarding/)**, then browse copy-paste starters (rules, GitHub Action, MCP setup, Docker) in [`examples/`](https://github.com/vanducng/miu-cr/tree/main/examples).
